# LangGraph 学习检查清单

> 系统化的学习路径,帮助你掌握 LangGraph 的每个核心概念

---

## 使用说明

本检查清单对应《LangGraph 核心概念与工程实践指南》的15个章节。每章包含:

- **学习目标**: 你应该掌握的核心知识点
- **实践任务**: 动手练习以巩固理解
- **自查问题**: 验证学习成果

建议:
1. 按顺序学习,每章完成后打勾
2. 不要跳过实践任务,动手是最好的学习方式
3. 自查问题回答不上来时,回到对应章节重新学习

---

## 第一篇: 基础知识篇

### 第 1 章: LangGraph 是什么

**学习目标**
- [ ] 理解为什么需要 LangGraph(Chain 的局限性)
- [ ] 掌握 LangGraph 的核心定位和价值主张
- [ ] 了解状态图(StateGraph)的基本概念
- [ ] 理解 LangGraph 与传统工作流引擎的区别

**实践任务**
- [ ] 画出一个简单场景(如客服助手)的工作流图
- [ ] 识别出该流程中的节点、边和状态
- [ ] 对比用 Chain 和 LangGraph 实现的差异

**自查问题**
1. 什么场景下应该选择 LangGraph 而不是 Chain?
2. LangGraph 的三大核心价值是什么?
3. "有向状态图"中的"状态"指什么?

### 第 2 章: 核心抽象

**学习目标**
- [ ] 深入理解节点(Node)的定义和作用

- [ ] 理解状态(State)的设计原则
- [ ] 了解 START 和 END 的特殊含义

**实践任务**
- [ ] 实现一个简单的3节点图(如:输入→处理→输出)
- [ ] 为图添加条件分支(使用条件边)
- [ ] 添加一个循环边,并设置终止条件

**自查问题**
1. 节点函数的标准签名是什么?
2. 条件边和普通边的区别在哪里?
3. 如何防止无限循环?

**学习目标**
- [ ] 了解传统工作流引擎(Airflow、Temporal)的特点

- [ ] 掌握 LangGraph 的核心差异点
- [ ] 知道何时选择 LangGraph,何时选择传统引擎

**实践任务**
- [ ] 列出你熟悉的工作流场景,判断是否适合 LangGraph
- [ ] 对比一个数据管道场景用 Airflow 和 LangGraph 的实现差异

**自查问题**

## 学习建议

### 初学者路径(1-2周)
1. **第1-4章**(基础知识篇): 理解核心概念
2. **第5-6章**(状态和路由): 掌握基本技能
